Placement Policies

Placement policies are implemented as a memory allocation strategy, to help determine where in main memory should incoming processes be placed.
In non-paged systems, placement policy must maintain a list of the locations and sizes of all unallocated parts of memory (free list). Then, it is decided which free memory should be used, and update the free list after each insertion

First-fit placement algorithm

The main memory is scanned for the first chunk of memory in the free list that is big enough to accommodate the requesting process.

Best-fit placement algorithm

The chunk of memory in the free list which produce the smallest left-over is chosen to be allocated to the process.

Worst-fit placement algorithm

In worst fit approach is to locate largest available free memory partition so that such portion will produce the most left-over memory space. It is the reverse of best fit.
